## 7. Nota areaServed (input per schema.org LocalBusiness / Organization)

Distinzione critica per non mentire al motore di ricerca e ai buyer:

- **SEDE LEGALE/OPERATIVA (address)** = unico indirizzo fisico:
  - Via Aldo Sandulli 91, Roma, Italia — CAP 00156, `addressCountry: IT`.
  - Email `info@fortunsrl.com`. Telefono [DA CONFERMARE: telefono pubblico]. PEC 16828791000@impresa.italia.it.

- **MERCATI SERVITI (`areaServed`)** = NON è la sede. I 3 hub Rotterdam/Houston/Fujairah sono **hub operativi di trading (TSA)**, non sedi → vanno modellati come `areaServed` + eventuale `Place`/descrizione, **non** come `address` secondari né come filiali.
  - `areaServed`: copertura internazionale. Suggerito modellare come aree/regioni di mercato:
    - Rotterdam → area ARA (Amsterdam-Rotterdam-Antwerp), Europa nord-occidentale.
    - Houston → US Gulf Coast, Americhe.
    - Fujairah → Middle East / hub del Golfo.
    - + copertura `areaServed: "Worldwide"` / mercati internazionali (coerente con "Presenza globale" e "Operazioni sui mercati internazionali").

- **Schema type consigliato:** `Organization` (NON `LocalBusiness` puro: Fortun non serve walk-in locali, è un trader B2B internazionale). Eventuale `subOrganization`/`OrganizationRole` per gli hub se serve, ma più pulito tenerli come testo + `areaServed`.
- **Catalogo prodotti** → `Organization` con `makesOffer`/`hasOfferCatalog` sui 5 prodotti (Diesel EN590, Jet Fuel A1, LPG, Fuel Oil, Crude Oil) — descritti per nome standard, **senza specifiche numeriche inventate**.
- **Campi identità da popolare nello schema appena disponibili (NON inventare):** `legalName` Fortun S.r.l., `vatID` 16828791000, `identifier REA` RM-1678563, `foundingDate` 2022. Finché ignoti → omettere il campo, non riempirlo.

> Coerenza messaggio: il sito dichiara **una sede a Roma** + **mercati globali via 3 hub**. Mai presentare gli hub come "uffici" o "sedi" (sarebbe falso e dannoso per la fiducia del buyer diffidente, oltre che per i rich result).

- `naics 424710` = Petroleum Bulk Stations and Terminals / wholesale petroleum; coerente con trading, non upstream. Rimuovere se non si vuole esporre codici.
- `sameAs`: lasciare solo URL profili reali e attivi. Se non esiste LinkedIn, rimuovere l'intero array (un `sameAs` con placeholder pubblicato vale zero e abbassa la credibilità).
- `identifier` per REA/capitale è opzionale: se non si vogliono esporre, eliminare i `PropertyValue` relativi anziché lasciarli placeholder.

## 4. BreadcrumbList (opzionale)

Raccomandazione: per un sito single-page (sezioni con anchor) il BreadcrumbList NON ha senso e Google lo ignora/segnala — una breadcrumb a un solo livello non porta rich result. **Ometterlo** se l'architettura è single-page.

Includerlo SOLO se esistono pagine reali separate (es. `/prodotti`, `/contatti`, `/en`). In quel caso:

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://www.fortunsrl.com"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Prodotti",
      "item": "https://www.fortunsrl.com/prodotti"
    }
  ]
}
```

## Varianti per la versione /en

Cambia SOLO ciò che è legato a lingua/URL. `@id` dei nodi condivisi (Organization, logo) restano IDENTICI tra IT e EN: è la stessa entità, va referenziata, non duplicata.

**WebPage/WebSite EN** — la pagina inglese dichiara `inLanguage: "en"` e linka l'alternato:
- `url` → `https://www.fortunsrl.com/en`
- `inLanguage` → `"en"`
- mantenere `publisher` / `about` → `{ "@id": "https://www.fortunsrl.com/#organization" }` (NON cambiare l'@id).

**OfferCatalog EN** — duplicare con:
- `@id` → `https://www.fortunsrl.com/en/#products`
- `url` → `https://www.fortunsrl.com/en#products`
- `inLanguage` → `"en"`
- `name` → "Refined petroleum products — Fortun SRL" (già EN-friendly)
- `LPG (GPL)` → in EN usare solo `"LPG"` (togliere "GPL").
- `provider` → invariato `{ "@id": ".../#organization" }`.

**Organization** — NON duplicare l'entità. Un solo nodo `#organization` per tutto il dominio. Se serve esporre `name`/`description` localizzati, gestirli a livello di pagina (`WebPage`), non creando una seconda Organization.

**Hreflang (fuori dal JSON-LD, ma necessario)** — nel `<head>` di ogni pagina:
```html
<link rel="alternate" hreflang="it" href="https://www.fortunsrl.com/" />
<link rel="alternate" hreflang="en" href="https://www.fortunsrl.com/en" />
<link rel="alternate" hreflang="x-default" href="https://www.fortunsrl.com/en" />
```
